{frontmatter.what}  

- [sort()](../sort/) is mutable - it changes the original array
- toSorted() is immutable - it does not change the original array

why use toSorted instead of sort:
- This allows you to mutate a copy of an array without needing to clone the array first

<div class="copy-to-clipboard-wrap" id="code-sample-01">
<CopyToClipboard id="code-sample-01"></CopyToClipboard>
```js
// const list = ['orange', 'cantaloupe', 'dragon fruit', 'apple', 'lemon', 'banana'];
const list = ['travaillé', 'étudiant', 'café', 'école']
const sortedList = list.toSorted();

console.log('list', list);
// Expected Output: ['travaillé', 'étudiant', 'café', 'école']

console.log('sortedList', sortedList);
// Expected Output: ['café', 'travaillé', 'école', 'étudiant']

const compareLocale = (a, b) => a.localeCompare(b);
const localeSortedList = list.toSorted(compareLocale);
console.log('localeSortedList', localeSortedList);
// Expected Output: ['café', 'école', 'étudiant', 'travaillé']
```
